Note

Any Java clients connecting to JBoss on the depression server will need to set the org.jboss.mq.il.uil2.useServerHost Java system property set to true:

java -classpath ... -Dorg.jboss.mq.il.uil2.useServerHost=true

This is because of the way the depression server has been set up.

When a JMS client looks up the server using the InetAddress class, the server should return something like: dmpeintw5386/172.20.23.132. Instead depression returns depression/127.0.0.1. This causes the client to use 127.0.0.1 as the server address. The org.jboss.mq.il.uil2.useServerHost system property forces the client to use depression as the host name.
