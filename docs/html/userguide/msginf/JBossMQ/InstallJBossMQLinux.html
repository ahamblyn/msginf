Installing JBoss onto Linux

This document outlines the process to install JBoss into a user's home directory and run it as a service.

JBoss is an open source J2EE compliant application server written in Java.

Setting up the Java Environment

Add the following to your .bash_profile file to set up your Java environment:

JAVA_HOME=/usr/java/jdk1.5.0_03
export JAVA_HOME
PATH=/usr/java/jdk1.5.0_03/bin:"${PATH}"

Installing JBoss

The latest binary distribution jboss-4.0.1RC1.tar.gz can be found on the 172.20.20.59 FTP server. Login as anonymous and use your email address as the password.

FTP this file to your home directory.

Unpack this file:

$cd
$gunzip jboss-4.0.1RC1.tar.gz
$tar -xvf jboss-4.0.1RC1.tar

This will create an jboss-4.0.1RC1 directory in your home directory.

Configuring JBoss

As the development VPayables instance of Tomcat on depression is already running on ports 8080 and 8009, the instance of Tomcat that JBoss uses as a servlet container will need to be configured to use different ports.

Make the following changes to the ~/jboss-4.0.1RC1/server/default/deploy/jbossweb-tomcat50.sar/server.xml file to use ports 18080 and 18009:

Current Change To
<!-- A HTTP/1.1 Connector on port 8080 -->
<Connector port="8080" address="${jboss.bind.address}"		 <!-- A HTTP/1.1 Connector on port 18080 -->
<Connector port="18080" address="${jboss.bind.address}"
<!-- A AJP 1.3 Connector on port 8009 -->
<Connector port="8009" address="${jboss.bind.address}"		 <!-- A AJP 1.3 Connector on port 18009 -->
<Connector port="18009" address="${jboss.bind.address}"

Remove the windows batch files:

$cd ~/jboss-4.0.1RC1/bin
$rm *.bat

Edit ~/jboss-4.0.1RC1/bin/run.conf to change the JVM memory settings:

Current Change To
JAVA_OPTS="-server -Xms128m -Xmx128m" JAVA_OPTS="-server -Xms128m -Xmx512m"

Run JBoss:

$cd ~/jboss-4.0.1RC1/bin
$./run.sh

Install JBoss as a Service

Java Service Wrapper is used to run JBoss as a service. This allows JBoss to be started with a script which takes stop, start, and restart as parameters. More details can be found here.

The Java Service Wrapper binary wrapper_linux_3.1.2.tar.gz can be found on the 172.20.20.59 FTP server. Login as anonymous and use your email address as the password.

FTP this file to your home directory and unpack:

$cd
$gunzip wrapper_linux_3.1.2.tar.gz
$tar -xvf wrapper_linux_3.1.2.tar

Copy the ~/wrapper_linux_3.1.2/bin/wrapper and ~/wrapper_linux_3.1.2/bin/wrapper/src/bin/sh.script.in files to the ~/jboss-4.0.1RC1/bin directory:

$cd ~/jboss-4.0.1RC1/bin
$cp ~/wrapper_linux_3.1.2/bin/wrapper .
$cp ~/wrapper_linux_3.1.2/src/bin/sh.script.in .

Rename the sh.script.in file to jboss:

$cd ~/jboss-4.0.1RC1/bin
$mv sh.script.in jboss

Open the jboss script and set the APP_NAME and APP_LONG_NAME properties:

APP_NAME="JBoss"
APP_LONG_NAME="JBoss Application Server"

Set the jboss script to be executable:

$cd ~/jboss-4.0.1RC1/bin
$chmod u+x jboss

Copy the following files into the ~/jboss-4.0.1RC1/lib directory:

$cd ~/jboss-4.0.1RC1/lib
$cp ~/wrapper_linux_3.1.2/lib/libwrapper.so .
$cp ~/wrapper_linux_3.1.2/lib/wrapper.jar .

Create the ~/jboss-4.0.1RC1/conf directory:

$cd ~/jboss-4.0.1RC1
$mkdir conf

Copy the following file into the ~/jboss-4.0.1RC1/conf directory and rename:

$cd ~/jboss-4.0.1RC1/conf
$cp ~/wrapper_linux_3.1.2/src/conf/wrapper.conf.in .
$mv wrapper.conf.in wrapper.conf

Create a logs directory:

$cd ~/jboss-4.0.1RC1
$mkdir logs

Before the wrapper can be configured to launch JBoss, the full Java command created by the current ~/jboss-4.0.1RC1/bin/run.sh script is required.

Copy the ~/jboss-4.0.1RC1/bin/run.sh file to ~/jboss-4.0.1RC1/bin/run.sh.tmp.

Edit the ~/jboss-4.0.1RC1/bin/run.sh file to output this information.

Find the lines (at the end of the script):

do
# Execute the JVM
   "$JAVA" $JAVA_OPTS \
      -Djava.endorsed.dirs="$JBOSS_ENDORSED_DIRS" \
      -classpath "$JBOSS_CLASSPATH" \
      org.jboss.Main "$@"
   STATUS=$?
   # if it doesn't work, you may want to take a look at this:
   #    http://developer.java.sun.com/developer/bugParade/bugs/4465334.html
done

Change to:

do
# Execute the JVM
   echo "$JAVA $JAVA_OPTS \
      -Djava.endorsed.dirs=$JBOSS_ENDORSED_DIRS \
      -classpath $JBOSS_CLASSPATH \
      org.jboss.Main $@"
   STATUS=$?
   # if it doesn't work, you may want to take a look at this:
   #    http://developer.java.sun.com/developer/bugParade/bugs/4465334.html
done

Run the ~/jboss-4.0.1RC1/bin/run.sh script:

$cd ~/jboss-4.0.1RC1/bin
$./run.sh

This will output the full Java command:

/usr/java/j2sdk1.4.2/bin/java -server -Xms128m -Xmx512m -Dprogram.name=run.sh -Djava.endorsed.dirs=/home/alisdairh/jboss-4.0.1RC1/lib/endorsed -classpath /home/alisdairh/jboss-4.0.1RC1/bin/run.jar:/usr/java/j2sdk1.4.2/lib/tools.jar org.jboss.Main

Replace the original run.sh script:

$cd ~/jboss-4.0.1RC1/bin
$rm run.sh
$mv run.sh.tmp run.sh

Using the information provided by the echo command above, make the following changes to the ~/jboss-4.0.1RC1/conf/wrapper.conf file:

Current Change To
wrapper.logfile=../logs/wrapper.log wrapper.logfile=../logs/jboss.log
wrapper.java.command=java wrapper.java.command=/usr/java/jdk1.5.0_03/bin/java
#wrapper.java.additional.1= wrapper.java.additional.1=-Dprogram.name=run.sh
wrapper.java.additional.2=-Djava.endorsed.dirs=../lib/endorsed
wrapper.java.additional.3=-Djava.rmi.server.hostname=depression
#wrapper.java.maxmemory=64 wrapper.java.maxmemory=512
wrapper.java.classpath.1=../lib/wrapper.jar wrapper.java.classpath.1=../lib/wrapper.jar
wrapper.java.classpath.2=../conf
wrapper.java.classpath.3=../bin/run.jar
wrapper.java.classpath.4=/usr/java/jdk1.5.0_03/lib/tools.jar
wrapper.app.parameter.1=<YourMainClass> wrapper.app.parameter.1=org.jboss.Main

$cd ~/jboss-4.0.1RC1/bin
$./jboss console
