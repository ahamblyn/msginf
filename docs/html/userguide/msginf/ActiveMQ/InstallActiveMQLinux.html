Installing ActiveMQ 3.1 onto Linux

This document outlines the process to install ActiveMQ 3.1 into a user's home directory and run it as a service.

ActiveMQ is an open source JMS compliant messaging system written in Java.

Setting up the Java Environment

Add the following to your .bash_profile file to set up your Java environment:

JAVA_HOME=/usr/java/jdk1.5.0_03
export JAVA_HOME
PATH=/usr/java/jdk1.5.0_03/bin:"${PATH}"

Installing ActiveMQ 3.1

The latest binary distribution activemq-3.1.tar.gz can be found on the 172.20.20.59 FTP server. Login as anonymous and use your email address as the password.

FTP this file to your home directory.

Unpack this file:

$cd
$gunzip activemq-3.1.tar.gz
$tar -xvf activemq-3.1.tar.gz

This will create an activemq-3.1 directory in your home directory.

Configuring ActiveMQ 3.1

Some adjustments need to be made to make ActiveMQ run.

Open the ~/activemq-3.1/conf/activemq.xml file:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE beans PUBLIC  "-//ACTIVEMQ//DTD//EN" "http://activemq.org/dtd/activemq.dtd">
<beans>
 
  <!-- ==================================================================== -->
  <!-- ActiveMQ Broker Configuration -->
  <!-- ==================================================================== -->
  <broker>
    <connector>
      <tcpServerTransport uri="tcp://localhost:61616" backlog="1000" useAsyncSend="true" maxOutstandingMessages="50"/>
    </connector>

    <persistence>
      <cachePersistence>
        <journalPersistence directory="../var/journal">
          <jdbcPersistence dataSourceRef="derby-ds"/>

          <!-- Choose one of the following three configurationss to work with mysql-ds-->
          <!--
          <jdbcPersistence dataSourceRef="mysql-ds"
                         adapterClass="org.activemq.store.jdbc.adapter.BytesJDBCAdaptor"/>
          <jdbcPersistence dataSourceRef="mysql-ds"
                         adapterClass="org.activemq.store.jdbc.adapter.BlobJDBCAdaptor"/>
          <jdbcPersistence dataSourceRef="mysql-ds"
                         adapterClass="org.activemq.store.jdbc.adapter.StreamJDBCAdaptor"/>
          -->
        </journalPersistence>
      </cachePersistence>
    </persistence>
  </broker>

  <!-- ==================================================================== -->
  <!-- JDBC DataSource Configurations -->
  <!-- ==================================================================== -->

  <!-- The Derby Datasource that will be used by the Broker -->
  <bean id="derby-ds" class="org.apache.commons.dbcp.BasicDataSource" destroy-method="close">
    <property name="driverClassName">
      <value>org.apache.derby.jdbc.EmbeddedDriver</value>
    </property>
    <property name="url">
      <!-- Use a URL like 'jdbc:hsqldb:hsql://localhost:9001' if you want to connect to a remote hsqldb -->
      <value>jdbc:derby:derbydb;create=true</value>
    </property>
    <property name="username">
      <value></value>
    </property>
    <property name="password">
      <value></value>
    </property>
    <property name="poolPreparedStatements">
      <value>true</value>
    </property>
  </bean>

  <!-- The MYSQL Datasource that will be used by the Broker -->
  <bean id="mysql-ds"
    class="org.apache.commons.dbcp.BasicDataSource"
    destroy-method="close">
    <property name="driverClassName">
      <value>com.mysql.jdbc.Driver</value>
    </property>
    <property name="url">
      <value>jdbc:mysql://localhost/activemq</value>
    </property>
    <property name="username">
      <value>myname</value>
    </property>
    <property name="password">
      <value>mypassword</value>
    </property>
    <property name="poolPreparedStatements">
      <value>true</value>
    </property>
  </bean>

</beans>

Replace localhost:61616 with 0.0.0.0:61616. This allows clients to access the ActiveMQ server from other computers.

Replace <cachePersistence> with <vmPersistence/>. This replaces persistence message storage using the internal ActiveMQ message store with in memory persistence.

Install ActiveMQ 3.1 as a Service

Java Service Wrapper is used to run ActiveMQ as a service. This allows ActiveMQ to be started with a script which takes stop, start, and restart as parameters. More details can be found here.

The Java Service Wrapper binary wrapper_linux_3.1.2.tar.gz can be found on the 172.20.20.59 FTP server. Login as anonymous and use your email address as the password.

FTP this file to your home directory and unpack:

$cd
$gunzip wrapper_linux_3.1.2.tar.gz
$tar -xvf wrapper_linux_3.1.2.tar

Copy the ~/wrapper_linux_3.1.2/bin/wrapper and ~/wrapper_linux_3.1.2/bin/wrapper/src/bin/sh.script.in files to the ~/activemq-3.1/bin directory:

$cd ~/activemq-3.1/bin
$cp ~/wrapper_linux_3.1.2/bin/wrapper .
$cp ~/wrapper_linux_3.1.2/src/bin/sh.script.in .

Rename the sh.script.in file to mq:

$cd ~/activemq-3.1/bin
$mv sh.script.in mq

Edit the mq script and set the APP_NAME and APP_LONG_NAME properties:

APP_NAME="ActiveMQ"
APP_LONG_NAME="ActiveMQ Messaging System"

Set the mq script to be executable:

$cd ~/activemq-3.1/bin
$chmod u+x mq

Copy the following files into the ~/activemq-3.1/lib directory:

$cd ~/activemq-3.1/lib
$cp ~/wrapper_linux_3.1.2/lib/libwrapper.so .
$cp ~/wrapper_linux_3.1.2/lib/wrapper.jar .

Copy the following file into the ~/activemq-3.1/conf directory and rename:

$cd ~/activemq-3.1/conf
$cp ~/wrapper_linux_3.1.2/src/conf/wrapper.conf.in .
$mv wrapper.conf.in wrapper.conf

Create a logs directory:

$cd ~/activemq-3.1
$mkdir logs

Before the wrapper can be configured to launch ActiveMQ, the full Java command created by the current ~/activemq-3.1/bin/activemq script is required.

Copy the ~/activemq-3.1/bin/activemq file to ~/activemq-3.1/bin/activemq.tmp.

Edit the ~/activemq-3.1/bin/activemq file to output this information.

Find the line:

exec "$JAVACMD" $ACTIVEMQ_DEBUG_OPTS $ACTIVEMQ_OPTS -classpath "$LOCALCLASSPATH" -Dactivemq.home="${ACTIVEMQ_HOME}" org.activemq.spring.Main $ACTIVEMQ_ARGS "$@"

Change it to:

echo "$JAVACMD $ACTIVEMQ_DEBUG_OPTS $ACTIVEMQ_OPTS -classpath $LOCALCLASSPATH -Dactivemq.home=${ACTIVEMQ_HOME} org.activemq.spring.Main $ACTIVEMQ_ARGS $@"

Run the ~/activemq-3.1/bin/activemq script:

$cd ~/activemq-3.1/bin
$./activemq

To start ActiveMQ 3.1 as a service:

$cd ~/activemq-3.1/bin
$./mq start

The following output should be seen:

Starting ActiveMQ Messaging System...

To stop ActiveMQ 3.1:

$cd ~/activemq-3.1/bin
$./mq stop

The following output should be seen:

Stopping ActiveMQ Messaging System...
Stopped ActiveMQ Messaging System.

The output seen during the console test is logged in the ~/activemq-3.1/logs/activemq.log file.
