Installing Open JMS 0.7.6.1 onto Linux

This document outlines the process to install Open JMS 0.7.6.1 into a user's home directory and run it as a service.

Open JMS is an open source JMS compliant messaging system written in Java.

Setting up the Java Environment

Add the following to your .bash_profile file to set up your Java environment:

JAVA_HOME=/usr/java/jdk1.5.0_03
export JAVA_HOME
PATH=/usr/java/jdk1.5.0_03/bin:"${PATH}"

Installing Open JMS 0.7.6.1

The latest source distribution openjms-0.7.6.1.zip can be found on the 172.20.20.59 FTP server. Login as anonymous and use your email address as the password.

FTP this file to your home directory.

Unzip this file:

$cd
$unzip openjms-0.7.6.1.zip

This will create an openjms-0.7.6.1 directory in your home directory.

Configuring Open JMS 0.7.6.1

Add the following to your .bash_profile file to set up the OPENJMS_HOME environment variable:

OPENJMS_HOME=~/openjms-0.7.6.1
export OPENJMS_HOME

All of the JNDI objects and factories are defined in the ~/openjms-0.7.6.1/config/openjms.xml file.

Edit the ~/openjms-0.7.6.1/config/openjms.xml file until it looks like:

<?xml version="1.0"?>

<!-- OpenJMS configuration:
     . RMI connectors
     . JDBM persistency
     . embedded JNDI provider
     . embedded RMI registry, running on port 1099
     . preconfigured destinations

     NOTES: this configuration shows all configuration elements relevant
            when using an RMI connector.
 -->

<Configuration>

  <!-- Optional. This represents the default configuration  -->
  <ServerConfiguration host="depression" embeddedJNDI="true" />

  <TcpConfiguration port="3030" jndiPort="3035"/>

  <!-- Required when using an RMI connector -->
  <Connectors>
    <Connector scheme="tcp">
      <ConnectionFactories>
        <QueueConnectionFactory name="TCPQueueConnectionFactory"/>
        <TopicConnectionFactory name="TCPTopicConnectionFactory"/>
      </ConnectionFactories>
    </Connector>
    <Connector scheme="rmi">
      <ConnectionFactories>
        <QueueConnectionFactory name="JmsQueueConnectionFactory" />
        <TopicConnectionFactory name="JmsTopicConnectionFactory" />
      </ConnectionFactories>
    </Connector>
  </Connectors>

  <!-- Required  -->
  <DatabaseConfiguration>
    <JdbmDatabaseConfiguration name="openjms.db" />
  </DatabaseConfiguration>

  <!-- Required -->
  <AdminConfiguration script="${openjms.home}\bin\startup.bat" />

  <!-- Optional. If not specified, no destinations will be created -->
  <AdministeredDestinations>
    <AdministeredTopic name="topic1">
      <Subscriber name="sub1" />
      <Subscriber name="sub2" />
    </AdministeredTopic>

    <AdministeredQueue name="TestQueue" />
    <AdministeredQueue name="RequestQueue" />
    <AdministeredQueue name="ReplyQueue" />
    <AdministeredQueue name="DeadLetterQueue" />
  </AdministeredDestinations>

  <!-- Optional. If not specified, no users will be created -->
  <Users>
    <User name="admin" password="openjms" />
  </Users>

</Configuration>

Change the permissions of the server startup scripts:

$cd ~/openjms-0.7.6.1/bin
$chmod u+x *.sh

Run Open JMS:

$cd ~/openjms-0.7.6.1/bin
$./startup.sh
Using OPENJMS_HOME: /home/alisdairh/openjms-0.7.6.1
Using JAVA_HOME:    /usr/java/j2sdk1.4.2
OpenJMS 0.7.6.1
The OpenJMS Group. (C) 1999-2004. All rights reserved.
http://openjms.sourceforge.net

Ctrl-C to stop Open JMS.

Install Open JMS 0.7.6.1 as a Service

Java Service Wrapper is used to run Open JMS as a service. This allows Open JMS to be started with a script which takes stop, start, and restart as parameters. More details can be found here.

The Java Service Wrapper binary wrapper_linux_3.1.2.tar.gz can be found on the 172.20.20.59 FTP server. Login as anonymous and use your email address as the password.

FTP this file to your home directory and unpack:

$cd
$gunzip wrapper_linux_3.1.2.tar.gz
$tar -xvf wrapper_linux_3.1.2.tar

Create the Open JMS service directory:

$cd ~/openjms-0.7.6.1/
$mkdir service

Create the bin, conf, config, lib, and logs directories in the service directory:

$cd ~/openjms-0.7.6.1/service
$mkdir bin
$mkdir conf
$mkdir config
$mkdir lib
$mkdir logs

Copy the ~/wrapper_linux_3.1.2/bin/wrapper and ~/wrapper_linux_3.1.2/bin/wrapper/src/bin/sh.script.in files to the ~/openjms-0.7.6.1/service/bin directory:

$cd ~/openjms-0.7.6.1/service/bin
$cp ~/wrapper_linux_3.1.2/bin/wrapper .
$cp ~/wrapper_linux_3.1.2/src/bin/sh.script.in .

Rename the sh.script.in file to openjms:

$cd ~/openjms-0.7.6.1/service/bin
$mv sh.script.in openjms

Edit the openjms script and set the APP_NAME and APP_LONG_NAME properties:

APP_NAME="OPENJMS"
APP_LONG_NAME="OPENJMS Messaging System"

Set the openjms script to be executable:

$cd ~/openjms-0.7.6.1/service/bin
$chmod u+x openjms

Copy the following files into the ~/openjms-0.7.6.1/service/lib directory:

$cd ~/openjms-0.7.6.1/service/lib
$cp ~/wrapper_linux_3.1.2/lib/libwrapper.so .
$cp ~/wrapper_linux_3.1.2/lib/wrapper.jar .
$cp ~/openjms-0.7.6.1/lib/*.jar .

Copy the following files into the ~/openjms-0.7.6.1/service/conf directory and rename:

$cd ~/openjms-0.7.6.1/service/conf
$cp ~/wrapper_linux_3.1.2/src/conf/wrapper.conf.in .
$mv wrapper.conf.in wrapper.conf

Copy the following files into the ~/openjms-0.7.6.1/service/config directory:

$cd ~/openjms-0.7.6.1/service/config
$cp ~/openjms-0.7.6.1/config/*.xml .
$cp ~/openjms-0.7.6.1/config/*.policy .

Before the wrapper can be configured to launch Open JMS, the full Java command created by the current ~/openjms-0.7.6.1/bin/openjms.sh script is required.

Copy the ~/openjms-0.7.6.1/bin/openjms.sh file to ~/openjms-0.7.6.1/bin/openjms.sh.tmp.

Edit the ~/openjms-0.7.6.1/bin/openjms.sh file to output this information.

Find the line:

elif [ "$1" = "start" ] ; then

Change the "$_RUNJAVA" lines immediately after it to:

echo "$_RUNJAVA $JAVA_OPTS -Dopenjms.home=$OPENJMS_HOME \
      -classpath $CLASSPATH \
      -Djava.security.manager -Djava.security.policy=$POLICY_FILE \
      org.exolab.jms.server.JmsServer $@"

Run the ~/openjms-0.7.6.1/bin/startup.sh script which will call the openjms.sh script:

$cd ~/openjms-0.7.6.1/bin
$./startup.sh

This will output the full Java command:

/usr/java/j2sdk1.4.2/bin/java -Dopenjms.home=/home/alisdairh/openjms-0.7.6.1 -classpath :/home/alisdairh/openjms-0.7.6.1/lib/openjms-0.7.6.1.jar -Djava.security.manager -Djava.security.policy=/home/alisdairh/openjms-0.7.6.1/config/openjms.policy org.exolab.jms.server.JmsServer

Replace the original openjms.sh script:

$cd ~/openjms-0.7.6.1/bin
$rm openjms.sh
$mv openjms.sh.tmp openjms.sh

Using the information provided by the echo command above, make the following changes to the ~/openjms-0.7.6.1/service/conf/wrapper.conf file:

Current Change To
wrapper.logfile=../logs/wrapper.log wrapper.logfile=../logs/openjms.log
wrapper.java.command=java wrapper.java.command=/usr/java/jdk1.5.0_03/bin/java
#wrapper.java.maxmemory=64 wrapper.java.maxmemory=512
wrapper.java.classpath.1=../lib/wrapper.jar wrapper.java.classpath.1=../lib/wrapper.jar
wrapper.java.classpath.2=../conf
wrapper.java.classpath.3=../lib/openjms-0.7.6.1.jar
#wrapper.java.additional.1= wrapper.java.additional.1=-Dopenjms.home=..
wrapper.java.additional.2=-Djava.security.manager
wrapper.java.additional.3=-Djava.security.policy=../config/openjms.policy
wrapper.app.parameter.1=<YourMainClass> wrapper.app.parameter.1=org.exolab.jms.server.JmsServer

Start Open JMS 0.7.6.1 in console mode to test:

$cd ~/openjms-0.7.6.1/service/bin
$./openjms console

The following output should be seen:

Running OPENJMS Messaging System...
wrapper  | --> Wrapper Started as Console
wrapper  | Launching a JVM...
jvm 1    | Wrapper (Version 3.1.2) http://wrapper.tanukisoftware.org
jvm 1    |
jvm 1    | OpenJMS 0.7.6.1
jvm 1    | The OpenJMS Group. (C) 1999-2004. All rights reserved.
jvm 1    | http://openjms.sourceforge.net

Ctrl-C to shutdown Open JMS 0.7.6.1:

wrapper  | INT trapped.  Shutting down.
wrapper  | <-- Wrapper Stopped

To start Open JMS 0.7.6.1 as a service:

$cd ~/openjms-0.7.6.1/service/bin
$./openjms start

The following output should be seen:

Starting OPENJMS Messaging System...

To stop Open JMS 0.7.6.1:

$cd ~/openjms-0.7.6.1/service/bin
$./openjms stop

The following output should be seen:

Stopping OPENJMS Messaging System...
Stopped OPENJMS Messaging System.

The output seen during the console test is logged in the ~/openjms-0.7.6.1/service/logs/openjms.log file.
